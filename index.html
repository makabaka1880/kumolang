KumoLang

Making Correct Cheesecakes.

A purely imperative specification language that formalizes cheesecake synthesis through small-step operational semantics and correctness-by-construction guarantees. Originally developed as a joke research paper, KumoLang is nevertheless a fully functional language with a reference implementation in Rust.

Language Constructs

KumoLang has five statements. Each models a physical culinary operation in the small-step operational semantics.

skip
<skip_stmt> ::= skip | (skip)
The no-op. Acknowledges the inevitable passage of time in the kitchen. Evaluates to the identity transition on the store.
new-mixture
(new-mixture <id>)
Allocates a fresh mixture domain — a generative, isolated nominal type. Think of it as declaring a new culinary namespace: batter, glaze, filling.
prep
(prep <dest> <source>)
The monadic constructor. Lifts a raw ingredient from a mixture domain into an isolated container type. Cream cheese becomes c_cheese.
pour
(pour <dest> <src1> <src2>)
The endomorphic binary operator. Combines two containers within the same mixture domain. Cross-domain pours are rejected at verification time.
pour-out
(pour-out <dest> <container> <target>)
The type-lifting functor. Transitions container contents across nominal domains into a target mixture. The container is consumed — the transfer is irreversible, like emulsification.

Operational Semantics

KumoLang manipulates two primary entities through a strict sequential list-monad algebra. Execution guarantees a well-formed directed acyclic dependency graph by construction.

Rule Description
Stasis skip leaves the store unchanged. The kitchen state persists.
Domain-Alloc new-mixture generates a fresh domain token and binds it in the store. Each mixture is a unique, isolated namespace.
Ing-Prep prep constructs a container from a mixture domain, recording provenance. The container carries the domain token as its type tag.
Pour-In pour validates domain equality between both source containers, then merges their histories into the destination. Domain mismatch → verification failure. Cross-contamination is literally unrepresentable.
Pour-Out pour-out transfers container contents into the target mixture domain and empties the source container. The operation is irreversible — an egg, once emulsified, cannot be recovered.

Read the full formalization in the paper: KumoLang: A Formal Verification Framework for Cheesecake Synthesis (PDF).

Features

  • Lisp-like S-expression syntax
  • Small-step operational semantics
  • Explicit resource tracking
  • Provenance extraction from traces
  • Dependency graph generation
  • Allergen source tracing
  • Cross-contamination prevention

Installation & Usage

# Build the reference interpreter cargo build --release # Verify a recipe cargo run < recipe.kumo # Or via pipe cat recipe.kumo | cargo run

Limitations

Current verification guarantees do not extend to:

  • Flavor quality
  • Texture quality
  • Oven malfunction
  • User error
  • Existential disappointment
  • Programmer–manager conflicts
